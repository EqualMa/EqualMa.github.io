(void)的用法

​ 在变量或函数调用前加(void)，是一种偶尔可以见到的用法。那么，它到底起什么作用呢？ 编译一个函数： int add(int a, int b, int c){ return a+b; } 会得到一个警告：未使用的参数c。 如果想去掉这个警告，可以这样写：
​ int add(int a, int b, int c){ (void)c; return a+b; } 有时，在返回值类型不是void的函数调用中，也可以看见在前面加(void)的用法，如： (void)close(fd); 这说明： 1.你并不关心函数的返回值，因此忽略它。 2.这其实不是写给人看的，也不是写给编译器看的，而是为某些特定的静态代码检测工具准备的。 3. 很多大公司特别注重代码规范。在进行代码静态检查的时候，如果函数的返回值类型非void，而没有对函数调用的返回值进行处理，就会被认为违反规范。也就 是说，会把函数返回值是否被处理作为一项检测标准。如果我们强制性地在函数调用前加上(void)，就可以这种方式来通知检测器，该函数调用的返回值不是 被遗漏了，而是确实不需要处理，因此可以忽略这项检查。这就和我们在项目中要求排除编译器警告的道理是一样的。
