The Loop

The Loop is the default mechanism WordPress uses for outputting posts through a theme‚Äôs template files. How many posts are retrieved is determined by the number of posts to show per page defined in the Reading settings.

The Loop loops through each post retrieved for the current page one at a time and performs the action specified in your theme for each post (the action is to process HTML or PHP code written in The Loop).

You can use the Loop for a number of different things, for example to:

  • display post titles and excerpts on your blog‚Äôs homepage;
  • display the content and comments on a single post;
  • display the content on an individual page using template tags; and
  • display data from Custom Post Types and Custom Fields.

The Basic Loop

<?php if ( have_posts() ) : ?>
    <?php while ( have_posts() ) : the_post(); ?>
        ... Display post content
    <?php endwhile; ?>
<?php endif; ?>

This loop says that when there are posts, loop through and Display post content for each post.

Using The Loop

The Loop should be placed in index.php, and in any other templates which are used to display post information.

An extremely simple index.php file would look like:

<?php
    get_header();
    if ( have_posts() ) : while ( have_posts() ) : the_post();
        the_content();
    endwhile;
    else :
        _e( 'Sorry, no posts matched your criteria.', 'textdomain' );
    endif;
    get_sidebar();
    get_footer(); 
?>

What the Loop Can Display

The latest version of the following list can be found here

The Loop can display a number of different elements for each post.

For example, some common template tags used in many themes are:

  • next_post_link() ‚Äì a link to the post published chronologically after the current post
  • previous_post_link() ‚Äì a link to the post published chronologically before the current post
  • the_category() ‚Äì the category or categories associated with the post or page being viewed
  • the_author() ‚Äì the author of the post or page
  • the_content() ‚Äì the main content for a post or page
  • the_excerpt() ‚Äì the first 55 words of a post‚Äôs main content followed by an ellipsis (‚Ä¶) or read more link that goes to the full post. You may also use the ‚ÄúExcerpt‚Äù field of a post to customize the length of a particular excerpt.
  • the_ID() ‚Äì the ID for the post or page
  • the_meta() ‚Äì the custom fields associated with the post or page
  • the_shortlink() ‚Äì a link to the page or post using the url of the site and the ID of the post or page
  • the_tags() ‚Äì the tag or tags associated with the post
  • the_title() ‚Äì the title of the post or page
  • the_time() ‚Äì the time or date for the post or page. This can be customized using standard php date function formatting.

You can also use conditional tags, such as:

  • is_home() ‚Äì Returns true if the current page is the homepage
  • is_admin() ‚Äì Returns true if inside Administration Screen, false otherwise
  • is_single() ‚Äì Returns true if the page is currently displaying a single post
  • is_page() ‚Äì Returns true if the page is currently displaying a single page
  • is_page_template() ‚Äì Can be used to determine if a page is using a specific template, for example: is_page_template('about-page.php')
  • is_category() ‚Äì Returns true if page or post has the specified category, for example: is_category('news')
  • is_tag() ‚Äì Returns true if a page or post has the specified tag
  • is_author() ‚Äì Returns true if inside author‚Äôs archive page
  • is_search() ‚Äì Returns true if the current page is a search results page
  • is_404() ‚Äì Returns true if the current page does not exist
  • has_excerpt() ‚Äì Returns true if the post or page has an excerpt

Examples

Basic Examples

Blog Archive

<?php if ( have_posts() ) : while ( have_posts() ) : the_post(); ?>

        <h2><?php the_title(); ?></h2>
    <?php the_post_thumbnail(); ?>
    <?php the_excerpt(); ?>
<?php endwhile; else: ?>
    <?php _e( 'Sorry, no posts matched your criteria.', 'textdomain' ); ?>
<?php endif; ?>

The simple loop checks if there are any posts.

  • If there are, outputs each post‚Äôs title, thumbnail, and excerpt.
  • If no posts exists, it displays the message in parentheses.

Individual Post

In WordPress, each post has its own page, which displays the relevant information for that post. Template tags allow you to customize which information you want to display.

<?php if ( have_posts() ) : while ( have_posts() ) : the_post(); ?>

<h1><?php the_title(); ?></h1>
    <?php the_content(); ?>
<?php endwhile; else: ?>
    <?php _e( 'Sorry, no pages matched your criteria.', 'textdomain' ); ?>
<?php endif; ?>

The loop outputs the post‚Äôs title and content. You could use this example in a post or page template file to display the most basic information about the post. You could also customize this template to add more data to the post, for example the category.

Style Posts from Some Categories Differently

<?php // Start the Loop.?>

    <?php else : ?>

        // Display the post's title.

        // Display a link to other posts by this posts author.

Multiple Loops

Use one queries in multiple loops

You can use rewind_posts() to loop through the same query a second time. This is useful if you want to display the same query twice in different locations on a page.

// Start the main loop
<?php
    if ( have_posts() ) : while ( have_posts() ) : the_post();
        the_title();
    endwhile;
    endif;

    // Use rewind_posts() to use the query a second time.
    rewind_posts();

    // Start a new loop
    while ( have_posts() ) : the_post();
        the_content();
    endwhile;
?>

Creating secondary queries and loops

You will often want to create a secondary query to display different content on the template. For example, you might want to display two groups of posts on the same page, but do different things to each group.

A common example of this is displaying a single post with a list of posts from the same category below the single post.

<?php
    // The main query.
    if ( have_posts() ) : while ( have_posts() ) : the_post();
        the_title();                                                             
        the_content();                                                          
    endwhile;                                                                   
    else :                                                                      
        // When no posts are found, output this text.                           
        _e( 'Sorry, no posts matched your criteria.' );                         
    endif;                                                                       
    wp_reset_postdata();                                                        

    /*                                                                          
     * The secondary query. Note that you can use any category name here. In our example,
     * we use "example-category".                                               
     */                                                                        
    $secondary_query = new WP_Query( 'category_name=example-category' );        

    // The second loop. if ( $secondary_query->have_posts() ) 
    echo '<ul>';
    while ( $secondary_query->have_posts() ) :
        $secondary_query->the_post();
        echo '<li>' . get_the_title() . '</li>';
    endwhile;
    echo '</ul>';
    endif;
wp_reset_postdata();
?>

As you can see in the example above, we first display a regular loop. Then we define a new variable that uses WP_Query to query a specific category; in our case, we chose the example-category slug.

Note that the regular loop in the example above has one difference: it calls wp_reset_postdata() to reset the post data. Before you can use a second loop, you need to reset the post data. There are two ways to do this:

  1. By using the rewind_posts() function; or
  2. By creating new query objects.

Resetting multiple loops

It‚Äôs important when using multiple loops in a template that you reset them.

Not doing so can lead to unexpected results due to how data is stored and used within the global $post variable.

There are three main ways to reset the loop depending on the way they are called.

  • wp_reset_postdata()
  • wp_reset_query()
  • rewind_posts()

Using wp_reset_postdata()

Use wp_reset_postdata() when you are running custom or multiple loops with WP_Query.

This function restores the global $post variable to the current post in the main query.

To properly use this function, place the following code after any loops with WP_Query:

<?php wp_reset_postdata(); ?>

Using wp_reset_query()

Using wp_reset_query() restores the WP_Query and global $post data to the original main query.

You MUST use this function to reset your loop if you use query_posts() within your loop.

You can use it after custom loops with WP_Query because it actually calls wp_reset_postdata()when it runs. However, it‚Äôs best practice to use wp_reset_postdata() with any custom loops involving WP_Query.

query_posts() is not best practice and should be avoided if at all possible. Therefore, you shouldn‚Äôt have much use for wp_reset_query().

To properly use this function, place the following code after any loops with query_posts().

<?php wp_reset_query(); ?>
